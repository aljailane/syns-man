# نظام التحديث المبسط

هذا هو السلوك المعتمد الآن بشكل بسيط وواضح.

## 1) متى يعمل التحديث التلقائي؟

- يعمل فقط في النسخة المبنية (Packaged).
- يعمل حاليًا لإصدارات Windows فقط.
- في وضع التطوير `npm start` يكون التحديث معطلًا طبيعيًا.

## 2) كيف يعمل داخل التطبيق؟

1. عند فتح التطبيق: فحص تحديث واحد تلقائيًا.
2. من صفحة About: زر `Check for Updates` للفحص اليدوي في أي وقت.
3. إذا وُجد تحديث: يبدأ التنزيل تلقائيًا.
4. بعد اكتمال التنزيل: يتفعّل زر `Install Update` لإعادة التشغيل والتثبيت.

## 3) متطلبات نجاح التحديث

في GitHub Release ارفع ملفات Windows الأساسية:

- ملف Setup (`.exe`)
- ملف blockmap (`.blockmap`)
- ملف `latest.yml`

## 4) لماذا هذا أبسط؟

- لا يوجد فحص دوري متكرر في الخلفية.
- لا يوجد فحوصات متوازية قد تسبب تعارضًا.
- رسائل الحالة أوضح عند عدم دعم المنصة أو وضع التطوير.

## 5) استكشاف سريع

- لا يظهر تحديث: تأكد أن رقم الإصدار في Release أعلى من النسخة الحالية.
- تظهر حالة Unsupported: هذا طبيعي خارج Windows.
- يظهر خطأ Release: تأكد وجود `latest.yml` وملفات Setup/Blockmap.
