# طرق التحديث (Update Methods)

هذه الصفحة تشرح كيف يعمل تحديث SYNS Man، وما هي طرقه.

## 1) التحديث التلقائي داخل التطبيق

يعتمد على `electron-updater` عبر GitHub Releases.

تدفق العمل:

1. التطبيق يفحص وجود تحديث.
2. عند وجود إصدار جديد يبدأ تنزيله.
3. يظهر التقدم في واجهة About.
4. بعد الاكتمال يظهر زر Install Update.

## 2) التحديث اليدوي للمستخدم

يمكن للمستخدم تحميل الإصدار الجديد من Releases وتثبيته مباشرة.

- ملف Setup للتثبيت التقليدي.
- ملف Portable للتشغيل المباشر.

## 3) التحديث أثناء التطوير

أثناء `npm start` يكون التطبيق في وضع development،
لذلك التحديث التلقائي يكون معطلًا (طبيعي).

## 4) متطلبات نجاح التحديث التلقائي

في صفحة Release يجب رفع:

- ملف Setup
- ملف blockmap
- ملف `latest.yml`

بدون هذه الملفات قد يفشل اكتشاف/تنزيل التحديث.

## 5) استكشاف مشاكل التحديث

### لا يوجد تحديث رغم وجود Release

تحقق من:

- رقم النسخة (`version`) أعلى من النسخة الحالية.
- وجود ملفات التحديث كاملة.
- صحة رابط المستودع في `build.publish`.

### التحديث يُكتشف لكن لا يُثبت

تحقق من:

- اكتمال التنزيل.
- الضغط على Install Update.
- صلاحيات النظام/مكافحة الفيروسات.

## 6) أفضل ممارسة

- حافظ على `CHANGELOG.md` محدث.
- ارفع إصدارات متدرجة (Patch/Minor/Major).
- اختبر التحديث على جهاز منفصل قبل نشر عام.
