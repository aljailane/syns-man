طرق التحديث

1) متى يعمل التحديث التلقائي؟

  • يعمل فقط في النسخة المبنية (Packaged).
  • مدعوم حاليًا لإصدارات Windows فقط.
  • في وضع npm start يكون التحديث معطلًا بشكل طبيعي.

2) آلية التحديث داخل التطبيق (بشكل مبسط)

  1. فحص تلقائي واحد عند تشغيل التطبيق.
  2. فحص يدوي عبر زر Check for Updates.
  3. التنزيل يبدأ تلقائيًا عند وجود إصدار جديد.
  4. بعد اكتمال التنزيل يتفعّل زر Install Update.

3) الملفات المطلوبة لنجاح التحديث التلقائي

  • Setup EXE
  • blockmap
  • latest.yml

4) لماذا النظام أبسط الآن؟

  • لا يوجد فحص دوري متكرر في الخلفية.
  • لا توجد فحوصات متوازية قد تسبب تعارضًا.
  • رسائل الحالة أوضح عند عدم الدعم أو وضع التطوير.

5) استكشاف سريع

  • لا يظهر تحديث: تأكد أن رقم الإصدار الجديد أعلى من الحالي.
  • ظهور حالة Unsupported: هذا طبيعي خارج Windows.
  • فشل الفحص: تحقق من وجود latest.yml وملفات Setup/Blockmap في Release.
